# Curated Media Persistence — Design

**Date:** 2026-06-09
**Status:** Approved by Arshia (Approach A — snapshot pipeline)
**Repos involved:** `proslync-mobile-v1.1` (app code, bundled images, manifest, script), `proslync-web-v1.1` (public GitHub repo hosting curated videos via jsdelivr)

## Problem

Banner videos, dashboard cover media, custom section logos, and profile photos set in the app are stored as device-local `file://` URIs (FileSystem `documentDirectory` + AsyncStorage). They can never reach a TestFlight device, and a reinstall wipes them. The avatar flow is worse: it tries to upload through `/api/files/avatar/presigned-url`, which does not exist on the VPS backend (file endpoints are still a proposed-endpoints item for Trajan).

## Goal

Media that Arshia sets in the simulator becomes **the app's content**: baked into every build, identical on every device and every TestFlight install, permanently. Local per-device overrides (the current picker behavior) keep working on top.

Decisions made during brainstorming:

1. Curation will be done fresh in the sim once this mechanism exists (no existing media to migrate — all containers verified empty on 2026-06-09).
2. Scope is app-wide content, not per-user synced data ("everyone, it IS the app").
3. Media updates ship with each build (no runtime remote-manifest fetching).

## Architecture overview

```
[Arshia curates in sim] → AsyncStorage + Documents/proslync-media/   (existing pattern)
        │
        ▼  npm run snapshot-media        (new script, runs on Mac)
[images → mobile repo assets/media/curated/  (bundled)]
[videos → web repo public/videos/curated/    (pushed, SHA-pinned jsdelivr URLs)]
        │
        ▼  generates
lib/media/curated-manifest.ts            (committed to mobile repo)
        │
        ▼  every build, every device
slot resolution: local override → curated default → legacy fallback
```

## 1. Media slots

A **slot** is a stable string naming one customizable media surface. Slots mirror the existing AsyncStorage keys:

| Slot | Surface | Code location today | Picker today |
|---|---|---|---|
| `profile-banner` | Athlete profile banner | `app/(tabs)/profile.tsx` (~lines 1842–1985), key `proslync:profile:bannerVideo:v1` | video only |
| `coach-banner` | Coach profile banner | `components/coach/coach-profile.tsx` (~lines 164–300), key `proslync:coachprofile:bannerVideo:v1` | video only |
| `cover-{sectionId}` | Home dashboard section covers | `app/(tabs)/index.tsx`, key `proslync:home:coverMedia:v2` (`{ [sectionId]: { uri, type } }`) | image or video |
| `logo-{sectionId}` | Home custom section logos | `app/(tabs)/index.tsx`, key `proslync:home:customLogos:v1` (`{ [sectionId]: uri }`) | image |
| `profile-avatar` | Profile photo / avatar | `app/edit-profile.tsx` (picker), displayed in `app/(tabs)/profile.tsx` via `user?.avatar?.url` | image (currently broken: backend upload) |

### Changes to slots

- **`profile-avatar` gets the banner treatment.** The edit-profile picker stops calling the nonexistent presigned-URL flow when the backend lacks it and instead persists locally: copy to `Documents/proslync-media/profile-avatar/`, store URI under new key `proslync:profile:avatar:v1`. Display resolution in profile tab and edit-profile: local override → curated → `user?.avatar?.url` → legacy fallback.
- **Banner pickers accept images too.** `profile-banner` and `coach-banner` pickers change from `MediaTypeOptions.Videos` to `MediaTypeOptions.All`. Persisted value gains a type, stored as JSON `{ uri, type: 'image' | 'video' }` under new versioned keys `proslync:profile:banner:v2` and `proslync:coachprofile:banner:v2` (hydration migrates v1 string values to `{ uri, type: 'video' }`).

## 2. Curated manifest

Generated file, committed to the mobile repo: `lib/media/curated-manifest.ts`.

```ts
// AUTO-GENERATED by scripts/snapshot-media.mjs — do not edit by hand.
import type { ImageSourcePropType } from 'react-native';

export type CuratedMediaEntry =
  | { type: 'image'; source: ImageSourcePropType }   // bundled
  | { type: 'video'; uri: string };                  // hosted (SHA-pinned CDN)

export const CURATED_MEDIA: Record<string, CuratedMediaEntry> = {
  'profile-banner': {
    type: 'video',
    uri: 'https://cdn.jsdelivr.net/gh/arshiarahnavard7-sys/proslync-web-v1.1@<full-commit-sha>/public/videos/curated/profile-banner.mp4',
  },
  'cover-ncaab': {
    type: 'image',
    source: require('../../assets/media/curated/cover-ncaab.jpg'),
  },
  // ...one entry per curated slot
};
```

Rules:

- **Images are bundled** (`require()` with literal static paths so Metro packs them). Offline-safe, no CDN dependency, no AVPlayer involvement.
- **Videos are hosted, never bundled** — bundled videos break expo-video's AVPlayer in production iOS .ipa builds (documented in web repo commit `c23f1f8`). URLs are jsdelivr with a **full commit SHA** (`@<sha>`), which jsdelivr treats as immutable and caches permanently. Verified 2026-06-09: jsdelivr serves the web repo's existing videos with HTTP 206 range support (required by AVPlayer); repo is PUBLIC.
- An empty manifest (`{}`) is valid — the app must work before the first snapshot ever runs.

## 3. Resolver

New hand-written module `lib/media/resolve-media.ts` exporting one hook used by every slot surface:

```ts
useSlotMedia(slot: string, localOverride: { uri: string; type: 'image' | 'video' } | null)
  → { kind: 'local' | 'curated' | 'none';
      type: 'image' | 'video';
      uri?: string;                       // local file or hosted video
      source?: ImageSourcePropType }      // bundled curated image
```

Resolution order:

1. **Local override** — the screen's existing AsyncStorage-hydrated value, but only if `FileSystem.getInfoAsync(uri)` confirms the file still exists. If the file is gone (orphaned pointer after reinstall), the hook clears the stale AsyncStorage key and falls through. (Existence check applies to `file://` URIs only.)
2. **Curated default** — `CURATED_MEDIA[slot]` if present.
3. **`none`** — caller renders its legacy fallback (current bundled PNGs / unsplash URLs / `user?.avatar?.url` for the avatar), exactly as today.

Screens keep their own picker + AsyncStorage hydration code (minimal diff); the only change at render sites is replacing direct `bannerVideo ? <VideoView/> : <legacy/>` branching with the resolver's output.

## 4. Snapshot script

`scripts/snapshot-media.mjs` (mobile repo), wired as `npm run snapshot-media`. Plain Node (nvm node v24), no new dependencies. Steps:

1. **Locate container:** `xcrun simctl get_app_container booted com.proslync.app data`. Error if no booted sim or app not installed.
2. **Read state:** parse `Library/Application Support/com.proslync.app/RCTAsyncLocalStorage_V1/manifest.json` (plus sibling per-key files for values >1 KB, which AsyncStorage stores separately). Collect all five slot families' values.
3. **Stage media:**
   - images → `assets/media/curated/<slot>.<ext>` (mobile repo)
   - videos → `public/videos/curated/<slot>.<ext>` (web repo working tree at `~/Desktop/proslync-web-v1.1`)
   - Extension whitelist: jpg/jpeg/png/webp/heic for images (heic converted to jpg via `sips` — heic renders unreliably in RN), mp4/mov for videos.
4. **Guardrails:** any video >20 MB (jsdelivr per-file limit) fails with an ffmpeg compression hint. Warn (not fail) on images >2 MB.
5. **Publish videos:** in the web repo — `git add public/videos/curated/`, commit (`chore(media): curated mobile media snapshot YYYY-MM-DD`), push to origin on whatever branch the web repo currently has checked out (SHA-pinned URLs don't care about branches once the commit is reachable on the remote), capture the **full commit SHA**. Skip commit/push if no video changes.
6. **Verify CDN:** for each video URL, poll `https://cdn.jsdelivr.net/gh/<owner>/<repo>@<sha>/<path>` with a range request until HTTP 206 (timeout 90 s; on timeout, print the URL and exit nonzero — manifest is still written, re-running the script re-verifies).
7. **Regenerate manifest:** merge new entries over the existing `CURATED_MEDIA` (parse the current file's entries; untouched slots keep their previous values — re-curating one banner never wipes the rest). Write `lib/media/curated-manifest.ts` with literal `require()` paths and SHA-pinned URLs.
8. **Report:** print the changed files in the mobile repo and the exact `git add … && git commit` to run (the script does NOT commit the mobile repo — Arshia reviews and commits).
9. **`--dry-run`:** prints everything it would do (slots found, files staged, sizes, URLs) without writing, committing, or pushing.

## 5. Error handling (runtime)

- **Hosted video fails to load** (offline, CDN hiccup): expo-video error → surface falls back to the curated image for that slot if one exists, else the legacy fallback. No black boxes, no spinners-forever.
- **Orphaned local overrides** self-heal (resolver clears stale keys, curated default shows).
- **Avatar:** if backend file endpoints appear later and `user.avatar.url` is set, it still participates in the fallback chain (after curated, since curated is the intentional demo look).

## 6. Testing & verification

1. `npx tsc --noEmit` clean after each implementation step.
2. **Authoring flow:** in sim — set a banner video, a cover image, a logo, an avatar → all render; relaunch app → all persist locally.
3. **Snapshot flow:** `npm run snapshot-media --dry-run` shows the four slots; real run produces manifest + assets, web repo pushed, CDN URLs return 206 (`curl -r 0-100 -o /dev/null -w '%{http_code}'`).
4. **The TestFlight simulation (the point of it all):** `xcrun simctl uninstall booted com.proslync.app` → rebuild/reinstall → fresh install shows the full curation with zero local state.
5. **Merge safety:** re-curate ONE slot, re-snapshot → other manifest entries unchanged (git diff shows one entry + one asset).
6. **Orphan healing:** delete a media file from the container but leave its AsyncStorage key → screen shows curated default, key gets cleaned.

## 7. Snapshot from a physical iPhone

Pass `--device` (or `--device <name-or-udid>`) to snapshot media curated on a cable-connected iPhone instead of the simulator:

```bash
npm run snapshot-media -- --device           # auto-pick the single paired device
npm run snapshot-media -- --device --dry-run # preview without writing
```

**Dev-signed builds only** — the container export (`xcrun devicectl device copy from`) requires a development-provisioned app. TestFlight and App Store builds are not eligible. The rest of the pipeline (AsyncStorage parsing, slot resolution, image/video staging, manifest generation) is identical to the simulator path: `exportDeviceContainer` copies the two relevant subtrees into a local temp dir laid out the same way, so `resolveContainerPath` re-anchors the phone's `file://` URIs onto the temp dir automatically.

## 8. Out of scope / follow-ups

- **Per-user media sync** across accounts/devices — requires VPS `/api/files/*` (presigned upload) + user profile media fields. Blocked on Trajan. This design layers under it: server-backed media would become resolution step 0 ahead of local override. Follow-up: write `proposed-endpoints/` spec in `proslync-backend-staging` for profile media fields.
- **In-app "Publish curation" button** (Approach C) — dev-mode one-tap alternative to running the script; can reuse the script's logic later if desired.
- Android: not targeted; nothing in the design is iOS-only except the snapshot script's `simctl` usage.
